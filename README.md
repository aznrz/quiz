# 🍥 Коноха & Бриджертоны Quiz (MS Cert PWA) 👑

[![Live Website](https://img.shields.io/badge/Live-MS--Cert%20Quiz-orange?style=for-the-badge&logo=cloudflare&logoColor=white)](https://naruto.ms-cert.workers.dev)
[![PWA Powered](https://img.shields.io/badge/PWA-Offline%20Support-blueviolet?style=for-the-badge&logo=pwa&logoColor=white)](#)
[![Firebase Integrated](https://img.shields.io/badge/Firebase-Integrated-yellow?style=for-the-badge&logo=firebase&logoColor=black)](#)

Современное, быстрое и визуально впечатляющее PWA-приложение (Progressive Web App) для тестирования знаний по вселенным аниме **«Наруто»** и сериала **«Бриджертоны»**.

Проект спроектирован как лёгкое статическое SPA-приложение, сочетающее серверную отказоустойчивость **Cloudflare Workers** и гибкость баз данных **Firebase (Firestore)** без необходимости развёртывания дорогостоящих серверов.

---

## 🌟 Ключевые возможности

* 🎌 **Две полноценные вселенные:**
  * **NARUTO Quiz** 🍥 (50 вопросов, от легких про Команду 7 до сложных техник запечатывания).
  * **BRIDGERTON Quiz** 👑 (30 вопросов, охватывающих сюжеты 1, 2 и 3 сезонов, балы, семейные тайны и интриги Леди Уислдаун).
* 📦 **Интеллектуальный движок интервального повторения (Leitner System):** Вопросы, в которых допущены ошибки, возвращаются в первый бокс для повторения на следующий день. Правильные ответы постепенно продвигают вопрос до Бокса 5 (полное освоение).
* 🔀 **Безопасное динамическое перемешивание ответов:** Уникальный алгоритм автоматической конвертации строковых вариантов ответов в типизированные объекты `{ key, text }` на этапе загрузки. Это предотвращает рассинхронизацию ключей при перемешивании и обеспечивает 100% точность оценивания.
* 📈 **Детальная статистика и Readiness:** Графики активности (активность за последние 14 дней, тепловая карта попыток), расчет процента готовности к экзамену («Readiness Score») с использованием Wilson-интервалов доверия.
* 📷 **Премиальные иллюстрации:** Встроенные аниме-портреты и винтажные Regency-эстетичные картины, генерируемые искусственным интеллектом под каждый вопрос.

## 🛠️ Стек технологий

* **Хостинг статики и Worker-слой:** Cloudflare Workers (современный быстрый рендеринг и глобальная сеть дистрибуции).
* **База данных и прогресс:** Firebase Cloud (Google Authentication + Firestore Database).
* **Движок фронтенда:** Чистый Vanilla JavaScript ES6 (SPA-архитектура, реактивное состояние).
* **Стилизация:** Кастомный CSS с адаптивным темным стеклянным дизайном (Glassmorphism), плавными микроанимациями и глубокой палитрой акцентных цветов.
* **Спецификация базы вопросов:** `data/questions.v2.json` (единый источник данных).

## 🚀 Быстрый старт (Разработка)

Для запуска локального сервера разработки выполните команду в корневой папке проекта:

```bash
# С использованием Node.js (http-server)
npx http-server -p 8080 -c-1

# Либо с помощью стандартного модуля Python
python -m http.server 8080
```

После этого откройте приложение в браузере: [http://localhost:8080](http://localhost:8080).

## ✍️ Добавление и изменение вопросов

Все вопросы хранятся в файле [`data/questions.v2.json`](file:///c:/Users/Aziz/GITHUB/naruto/data/questions.v2.json). 

Пример структуры вопроса:
```json
{
  "id": "nrt-051",
  "exam_code": "BRIDGERTON",
  "section": "easy",
  "version": "gen1",
  "options": [
    "Дафна Бриджертон",
    "Элоиза Бриджертон",
    "Пенелопа Фезерингтон",
    "Леди Дэнбери"
  ],
  "correct": 2,
  "explanation": "Личность таинственной Леди Уислдаун раскрывается зрителям в конце первого сезона — это Пенелопа Фезерингтон.",
  "prompt": "Кто является таинственной Леди Уислдаун в сериале «Бриджертоны»?",
  "images": [
    "assets/lady_whistledown.png"
  ]
}
```

### Спецификация полей:
* **`section`** — уровень сложности вопроса: `easy` (Легкий), `medium` (Средний), `hard` (Сложный).
* **`correct`** — 0-индексированный порядковый номер правильного ответа из списка `options` (в данном случае `2` указывает на `"Пенелопа Фезерингтон"`).
* **`prompt`** — текст вопроса.
* **`images`** — необязательный массив относительных путей к иллюстрациям вопроса.

> [!TIP]
> При добавлении новых вопросов вам больше не нужно вручную прописывать `correct_answers` в виде массива букв вроде `["C"]`! Движок нормализации на лету преобразует простые строковые варианты ответов в объекты с уникальными ключами и сгенерирует правильный буквенный маркер для полной поддержки перемешивания!

## ⚡ Синхронизация и Деплой

Публикация изменений на боевой сайт выполняется с помощью терминала:

### 1. Синхронизация вопросов перед публикацией
Скопируйте актуальный файл базы вопросов в бандл облачных функций:
```bash
node functions/sync-data.js
```

### 2. Деплой статики и PWA-слоя в Cloudflare
Опубликуйте изменения во фронтенде:
```bash
npx wrangler deploy
```

### 3. Деплой Firestore-правил и индексов
Обновите правила безопасности базы данных Firebase (при изменении `firestore.rules`):
```bash
firebase deploy --only firestore,hosting
```
