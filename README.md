# Коноха Quiz (Naruto Quiz) 🍥

Красивое и быстрое веб-приложение (PWA) для тестирования знаний по вселенной аниме **«Наруто»** и сериалу **«Бриджертоны»**, созданное для друзей и фанатов. 

Проект разработан как легкое статическое SPA-приложение, полностью совместимое с бесплатным тарифом **Firebase Spark**.

Живой сайт: [https://naruto.ms-cert.workers.dev](https://naruto.ms-cert.workers.dev)

---

## 🛠️ Стек технологий

* **Хостинг:** Cloudflare Workers (статический хостинг PWA).
* **Авторизация и Прогресс:** Firebase (Google Auth + Firestore Database).
* **Архитектура:** Полностью статическая. Вопросы загружаются клиентом из локального JSON, а события и статистика пишутся напрямую в Firestore по защищенным правилам `firestore.rules`.
* **База данных вопросов:** `data/questions.v2.json` (60 вопросов на русском языке).
* **Дизайн:** Адаптивный темный интерфейс на чистом CSS (акцентные цвета — оранжевый `#f97316` и пламя Конохи).

## 📂 Структура репозитория

| Путь | Назначение |
|------|------------|
| `data/questions.v2.json` | Единый источник правды для вопросов викторины |
| `src/app.js` | Ядро логики приложения (проверка ответов, Leitner, интервалы) |
| `src/style.css` | Глобальные стили приложения (оранжево-красная палитра Наруто) |
| `src/branding.js` | Конфигурация имени сайта (`SITE_NAME = 'Naruto Quiz'`) |
| `src/config/exam-profiles.js` | Профиль викторины `NARUTO` (веса разделов, тайминги) |
| `src/engine/` | Алгоритмы расчета готовности (`readiness.js`) и рекомендаций |
| `src/ui/` | Компоненты UI (план обучения, отчеты тренера, карточки) |
| `index.html` | Точка входа SPA-приложения |
| `firestore.rules` | Безопасные правила Firestore для Spark-тарифа |
| `wrangler.jsonc` | Конфигурация деплоя в Cloudflare |

## 🚀 Локальный запуск (Разработка)

Для запуска локального сервера разработки выполните команду в корневой папке репозитория:

```bash
# Использование Node.js http-server
npx http-server -p 8080 -c-1

# Либо с помощью Python
python -m http.server 8080
```

После этого откройте в браузере: [http://localhost:8080](http://localhost:8080)

## 📦 Инструкция по редактированию контента

Все вопросы хранятся в файле [data/questions.v2.json](file:///c:/Users/AzizNazarov/GiHUB/naruto/data/questions.v2.json). 

Каждый вопрос имеет следующую структуру:
```json
{
  "id": "nrt-051",
  "exam_code": "NARUTO",
  "section": "easy",
  "version": "gen1",
  "options": [
    "Вариант A",
    "Вариант B",
    "Вариант C",
    "Вариант D"
  ],
  "correct": 2,
  "explanation": "Подробное текстовое пояснение, почему этот вариант верный.",
  "prompt": "Текст вопроса?"
}
```

* **`section`** — уровень сложности вопроса: `easy` (Лёгкие), `medium` (Средние), `hard` (Эксперт).
* **`correct`** — 0-индексированный номер правильного ответа (от 0 до 3).
* **`prompt`** — текст вопроса, выводимый пользователю.

## ⚡ Деплой на Cloudflare Workers

Публикация изменений на живом хостинге выполняется в два простых шага:

1. **Синхронизация локальных вопросов:**
   ```bash
   node functions/sync-data.js
   ```
2. **Деплой фронтенда на Cloudflare:**
   ```bash
   npx wrangler deploy
   ```

Деплой Firestore правил (при изменении `firestore.rules`):
```bash
firebase deploy --only firestore
```
